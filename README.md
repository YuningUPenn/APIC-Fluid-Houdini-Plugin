# APIC Fluid Solver — Houdini Plugin

A Houdini Digital Asset (HDA) implementing the **Affine Particle-In-Cell (APIC)** 
method for fluid simulation, based on:

> Jiang, C., Schroeder, C., Selle, A., Teran, J., and Stomakhin, A.  
> *The Affine Particle-In-Cell Method.*  
> ACM Transactions on Graphics (SIGGRAPH 2015), 34(4), Article 51.

---

## Build Instructions

### 1. Source Houdini environment

**Linux / macOS:**
```bash
source /opt/hfs21.0/houdini_setup
```

**Windows (PowerShell):**
```powershell
& "C:\Program Files\Side Effects Software\Houdini 21.0\bin\hcustom.exe" -s
```

### 2. Install Eigen

**Linux:**
```bash
sudo apt install libeigen3-dev
# or: vcpkg install eigen3
```

**macOS:**
```bash
brew install eigen
```

**Windows:**
```powershell
vcpkg install eigen3
```

### 3. Configure and build

```bash
cd apic_fluid
mkdir build && cd build

cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DHOUDINI_PATH=/opt/hfs21.0

cmake --build . --config Release -j$(nproc)
ctest --output-on-failure   # run unit tests
```

### 4. Install the DSO

Copy the built shared library to Houdini's DSO path:

```bash
# Linux
cp build/libSOP_ApicFluid.so $HOME/houdini21.0/dso/

# macOS
cp build/libSOP_ApicFluid.dylib $HOME/Library/Preferences/houdini/21.0/dso/

# Windows
copy build\Release\SOP_ApicFluid.dll %USERPROFILE%\Documents\houdini21.0\dso\
```

### 5. Create the HDA

Open Houdini, go to **Windows → Python Shell**, and run:

```python
exec(open('/path/to/apic_fluid/python/create_hda.py').read())
```

Then install the HDA via **Assets → Install Asset Library** and point it to
`hda/apic_fluid_solver.hda`.

## Quick Start

1. Create a geometry object in Houdini.
2. Inside it, add a **Sphere** SOP (this will be your fluid volume).
3. Add the **APIC Fluid Solver** SOP and connect the Sphere to input 0.
4. Set **Particle Separation** to 0.05 and **Grid Resolution** to 64.
5. Press **Simulate** in the parameter panel.
6. Connect a **Particle Fluid Surface** SOP to the output to generate a mesh.

---

## APIC Simulation Loop

Each substep executes:

```
1. P2G  — rasterise particles to grid
           m_i  += w_ip * m_p
           m_i*v_i += w_ip * m_p * (v_p + B_p * D_p^{-1} * (x_i - x_p))

2. Grid update  — apply gravity, enforce solid BC

3. Pressure solve  — PCG Poisson projection (Eigen)
           Laplacian(p) = (rho/dt) * div(v*)
           v = v* - (dt/rho) * grad(p)

4. G2P  — interpolate back to particles, update B
           v_p = sum_i  w_ip * v_i
           B_p = sum_i  w_ip * v_i * (x_i - x_p)^T

5. Advect  — forward Euler
           x_p += dt * v_p
```

---

## Transfer Methods

| Method | B matrix | Characteristics |
|---|---|---|
| **APIC** | Updated each step | Stable, low dissipation, angular momentum preserved |
| **FLIP** | Not used | Energetic but noisy; can be unstable |
| **PIC** | Zeroed | Very stable but highly dissipative |
| **Hybrid** | APIC path | Blend of APIC and PIC characteristics |

## Running Unit Tests

```bash
cd build
ctest -V
```

Tests validate:
- **Mass and linear momentum conservation** in P2G transfer
- **Angular momentum preservation** — APIC error significantly less than PIC
- **Pressure solver convergence** — divergence reduced by ≥ 90%

---

## Known Limitations (Alpha)

- Collocated grid (not MAC) — pressure projection is less accurate near boundaries
- No adaptive time-stepping (CFL not enforced automatically)
- Viscosity and surface tension parameters exist but explicit viscosity is not 
  yet wired into the grid update
- Collision SDF from Houdini geometry uses a box approximation; full VDB 
  sampling is planned for beta

## Final Version Update

Here's a sample workflow for using the APIC SOP node:

```
Sphere (or any input mesh) -> VDB from Polygon (use fog VDB) -> Scatter -> APIC Fluid Solver -> VDB from particles (use voxel size 0.03) -> VDB smooth (with 3-5 iterations) -> Convert VDB (Convert to Polygons)
```

Collider can be added to the Solver, but in order to see the collider you need to merge the collider with the output of the workflow above.
